# Clippings NotebookLM

[English](#english) · [한국어](#한국어)

---

## English

An Obsidian plugin that analyzes Clippings notes with NotebookLM AI and auto-generates a summary and PPTX presentation using NotebookLM's native slide engine.

### Features

- Auto-injects a **"Generate PPT with NotebookLM"** button on every note in your Clippings folder
- Three report modes:
  - 📋 **Detailed** — for subject-matter experts, 8–12 slides
  - ⚡ **Executive** — for C-level leaders, 5–8 slides
  - 🌱 **Easy** — for mixed / non-expert audiences, 5–8 slides
- AI summary inserted as a callout block at the top of the note
- PPTX generated by NotebookLM Studio (professional design, native quality)
- Optional NotebookLM logo removal via slide revision

### Prerequisites

**1. Install the Obsidian Web Clipper browser extension**

Install the [Obsidian Web Clipper](https://obsidian.md/clipper) extension for your browser (Chrome, Firefox, Safari, or Edge). This extension lets you save web pages directly into your Obsidian vault as Clippings notes.

> [!TIP]
> After installing, configure the extension to save clips into your `Clippings` folder.

**2. Install notebooklm-mcp-cli**

```bash
uv tool install notebooklm-mcp-cli
```

> [!TIP]
> If you don't have `uv`, install it first: https://docs.astral.sh/uv/getting-started/installation/

**3. Log in to NotebookLM**

```bash
nlm login
```

A browser window will open for Google OAuth sign-in.

### Installation

#### Option A — BRAT (recommended, auto-updates)

1. Install the [BRAT](https://github.com/TfTHacker/obsidian42-brat) community plugin.
2. Open **Settings → BRAT → Add Beta Plugin**.
3. Enter the repository URL:
   ```
   https://github.com/wikilog/obsidian-clippings-notebooklm
   ```
4. Click **Add Plugin**, then enable **Clippings NotebookLM** in Community Plugins.

#### Option B — Manual

1. Download `main.js`, `manifest.json`, and `styles.css` from the [latest release](https://github.com/wikilog/obsidian-clippings-notebooklm/releases/latest).
2. Copy them into `<vault>/.obsidian/plugins/clippings-notebooklm/`.
3. Enable **Clippings NotebookLM** in **Settings → Community Plugins**.

### Setup

1. Go to **Settings → Clippings NotebookLM**.
2. Click **🌐 Login via Browser** and complete Google sign-in.
3. Click **Check Status** to confirm login.

### Usage

1. Use the **Obsidian Web Clipper** extension to save a web page into your Clippings folder.
2. Open the saved Markdown file inside your Clippings folder.
3. Click the **"Generate PPT with NotebookLM"** button at the top of the note.
4. Select a report mode — AI will generate a summary and a PPTX file automatically.

**Output example:**

```markdown
> [!summary] ⚡ Executive Report
> Key insight: ...
>
> 📎 **PPT:** [[Clippings/PDF/Article Title.pptx]]

# Article Title
(original content...)
```

### How It Works

The plugin uses NotebookLM's native slide engine (`nlm slides create`) instead of manually generating slides:

1. Creates a temporary NotebookLM notebook
2. Uploads the note content as a source
3. Queries NotebookLM for an AI summary
4. Calls `nlm slides create` with mode-specific format, length, and focus prompt
5. Optionally revises slide 1 to remove NotebookLM branding
6. Downloads the result as a PPTX file
7. Saves the file and inserts the summary + link into the note
8. Deletes the temporary notebook

### Troubleshooting

**`nlm` not found (PATH issue)**

Obsidian launched from Finder/Spotlight does not inherit your terminal `$PATH`. If the plugin cannot find `nlm`, enter the full path in **Settings → nlm CLI path**:

```
/Users/yourname/.local/bin/nlm
```

To find the path, run this in your terminal:

```bash
which nlm
```

**Login button does nothing**

Run `nlm login` directly in your terminal, then click **Check Status** in the plugin settings.

### Settings

| Setting | Default | Description |
|---------|---------|-------------|
| nlm CLI path | `nlm` | Path to the `nlm` executable |
| Remove NotebookLM branding | `true` | Revise slide 1 to remove the logo (adds ~1 min) |
| Clippings folder | `Clippings` | Folder where buttons are injected |
| Output subfolder | `PDF` | Subfolder inside Clippings for PPTX files |

## 한국어

Obsidian 플러그인 — Clippings 폴더의 마크다운 파일을 NotebookLM AI로 분석하여 요약 + PPTX 프레젠테이션을 자동 생성합니다. NotebookLM의 네이티브 슬라이드 엔진을 사용합니다.

### 기능

- Clippings 폴더의 모든 노트에 **"NotebookLM으로 PPT 만들기"** 버튼 자동 표시
- 3가지 리포트 모드:
  - 📋 **자세한 리포트** — 전문가 대상, 8–12 슬라이드
  - ⚡ **핵심 리포트** — C-레벨 임원 대상, 5–8 슬라이드
  - 🌱 **쉬운 리포트** — 비전문가 혼합 청중 대상, 5–8 슬라이드
- AI 요약이 노트 상단에 callout 블록으로 자동 삽입
- NotebookLM Studio로 생성된 PPTX (프로페셔널 디자인, 네이티브 품질)
- 선택적으로 NotebookLM 로고 제거 기능 제공

```markdown
> [!summary] ⚡ 핵심 리포트

```

### 작동 방식

플러그인은 직접 슬라이드를 조립하는 대신 NotebookLM의 네이티브 슬라이드 엔진(`nlm slides create`)을 사용합니다:

1. 임시 NotebookLM 노트북 생성
2. 노트 내용을 소스로 업로드
3. NotebookLM에 AI 요약 요청
4. 모드별 형식·길이·포커스 프롬프트로 `nlm slides create` 호출
5. 선택 시 1번 슬라이드를 수정하여 NotebookLM 브랜딩 제거
6. 결과물을 PPTX 파일로 다운로드
7. 파일 저장 및 노트에 요약 + 링크 삽입
8. 임시 노트북 삭제

### 문제 해결

**`nlm`을 찾을 수 없음 (PATH 문제)**

Finder나 Spotlight로 실행한 Obsidian은 터미널의 `$PATH`를 상속받지 않습니다. `nlm`을 찾지 못할 경우 **설정 → nlm CLI 경로** 에 전체 경로를 직접 입력하세요:

```
/Users/yourname/.local/bin/nlm
```

전체 경로는 터미널에서 다음 명령으로 확인할 수 있습니다:

```bash
which nlm
```

**로그인 버튼이 작동하지 않을 때**

터미널에서 `nlm login`을 직접 실행한 뒤, 플러그인 설정에서 **상태 확인** 버튼을 누르세요.

### 설정 옵션

| 설정 | 기본값 | 설명 |
|------|--------|------|
| nlm CLI 경로 | `nlm` | nlm 실행 파일 경로 |
| NotebookLM 브랜딩 제거 | `true` | 슬라이드 1번을 수정해 로고 제거 (약 1분 추가 소요) |
| Clippings 폴더 | `Clippings` | 버튼을 표시할 폴더 |
| 출력 서브폴더 | `PDF` | PPTX 저장 서브폴더 |
